# KinkLessons

🌐 Live Site: https://kinklessons.github.io/

KinkLessons is a collection of educational, exploratory, and self-reflection tools designed to help people better understand kink, communication, boundaries, and compatibility in consensual adult dynamics.

The project is built as a modular set of static web tools hosted via GitHub Pages.

---

## ✨ Projects

KinkLessons currently consists of four core projects:

### 📘 General Information Site
https://kinklessons.github.io/

A landing site designed for newcomers exploring kink and BDSM concepts for the first time. It provides:
- Introductory explanations
- Educational context for safe exploration
- Links to all related tools

---

### 📚 Kink Glossary
https://kinklessons.github.io/kink-glossary/

A structured glossary of kink-related terminology.

Includes:
- Common BDSM and kink definitions
- Contextual explanations of terms
- Educational framing for safe, consensual practice
- Continuously expanding terminology set

---

### ❤️ Kink Questionnaire
https://kinklessons.github.io/kink-questionnaire/

An interactive self-reflection tool designed to help users explore:
- Interests and preferences
- Boundaries and limits
- Compatibility with partners

Features:
- Scored questionnaire system
- Shareable results via URL encoding
- Local-only storage when no shared link is present
- Lightweight, privacy-focused design

---

### 🤝 Partner Compatibility Tool
https://kinklessons.github.io/kink-questionnaire/

A companion workflow for comparing responses between partners to better understand alignment, differences, and shared interests.

Includes:
- Side-by-side comparison logic
- Compatibility scoring
- Conversation-starting summaries
- Emphasis on communication over labels

---

## 🧠 Design Philosophy

KinkLessons is built around a few core principles:

- Education first — clarity over jargon  
- Consent-focused framing — always centered on communication and boundaries  
- Privacy-aware design — no unnecessary data storage  
- Self-reflection over labeling — tools are guides, not diagnoses  
- Accessibility — simple UI, readable structure, mobile-friendly layout  

---

## 🛠️ Tech Stack

- Static HTML / CSS / JavaScript  
- Vite (for questionnaire tooling)  
- GitHub Pages (hosting)  
- URL-encoded state sharing for questionnaire results  

---

## 🔐 Privacy

- No accounts required  
- No tracking of personal responses  
- Questionnaire data is stored locally unless explicitly encoded into a shareable link  
- No server-side storage of user inputs
